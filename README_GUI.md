# GUI-модуль (роль "Разработчик интерфейса и визуализации")

## Что внутри

Реализованы все пункты задачи 2 (главное окно, вкладки, таблицы данных,
таблица расписания, тепловая карта, статистика+диаграмма, экспорт PNG).

Новые файлы:
- `mainwindow.h/.cpp/.ui` — главное окно с тремя вкладками (Данные / Генерация / Результат).
- `datatab.h/.cpp` — вкладка "Данные": таблицы групп, преподавателей, аудиторий, дисциплин,
  кнопки Добавить/Удалить/Редактировать, импорт из JSON.
- `teacheravailabilitydialog.h/.cpp` — диалог редактирования доступности преподавателя (сетка день×пара).
- `subjectdialog.h/.cpp` — диалог добавления/редактирования дисциплины (выбор преподавателя и групп).
- `generationtab.h/.cpp` — вкладка "Генерация": кнопка запуска, прогресс-бар, журнал ошибок назначения.
- `resulttab.h/.cpp` — вкладка "Результат": таблица расписания с фильтром по группе и цветовой
  подсветкой, тепловая карта, статистика, диаграмма загрузки, экспорт CSV/JSON/PNG.
- `heatmapwidget.h/.cpp` — кастомный QWidget с paintEvent для тепловой карты.
- `barchartwidget.h/.cpp` — кастомная столбчатая диаграмма на QPainter (без QtCharts).
- `scheduleutils.h` — общие хелперы (названия дней, цвета).
- `scheduler.h/.cpp` — интерфейс `IScheduler` + `StubScheduler` (см. ниже).

## Важно для разработчика алгоритма

В файле `scheduler.h` есть интерфейс:

```cpp
class IScheduler {
public:
    virtual GenerationResult generate(const QVector<Group>&, const QVector<Teacher>&,
                                       const QVector<Room>&, const QVector<Subject>&) = 0;
};
```

Сейчас в `mainwindow.cpp` создаётся `StubScheduler` — упрощённая реализация
жадного алгоритма (она реально работает и сортирует занятия по принципу
"наиболее ограниченной переменной", как описано в техническом описании
проекта), но без полноценной обработки всех ограничений и оптимизаций.

Когда будет готов настоящий алгоритм — нужно:
1. Создать новый класс, наследующий `IScheduler`, и реализовать `generate(...)`.
2. В `mainwindow.cpp` заменить строку `m_scheduler = new StubScheduler();`
   на создание нового класса.
GUI менять не потребуется.

## Исправленный баг в datamanager.cpp

В исходном `datamanager.cpp` (не моя зона ответственности, но без этого GUI
не работал) была ошибка: `return true;` стоял в середине функции
`loadFromJson`, из-за чего парсинг аудиторий и дисциплин никогда не
выполнялся. Также в `exportScheduleToJson` отсутствовала закрывающая скобка,
из-за чего `exportStatisticsToJson` физически оказывался "внутри" другой
функции и код не мог скомпилироваться. Оба места исправлены, остальная
логика DataManager не менялась. Также добавлен CRUD-API (`addGroup`,
`updateGroup`, `removeGroup` и аналогичные для остальных сущностей) —
он нужен таблицам редактирования данных.

## Зависимости

Используется только `Qt6::Core` и `Qt6::Widgets` — никакие дополнительные
модули (QtCharts) подключать не нужно, диаграмма нарисована на QPainter.
